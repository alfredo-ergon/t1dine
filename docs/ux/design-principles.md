# Design Principles

- The primary action is meal creation, not database browsing.
- Optimise repeated meals to a few taps.
- Use plain language before clinical abbreviations.
- Always show units and source freshness.
- Colour is never the only warning signal.
- Support large text, screen readers, reduced motion, high contrast, and motor-accessible targets.
- Display uncertainty where it affects a decision.
- Avoid shame, judgement, and “good/bad food” framing.
- Make profile switching unmistakable.
- Preserve user control over photo upload, cloud sync, and sharing.
