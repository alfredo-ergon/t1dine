# ADR 0008: PortFIR / INSA Source Onboarding and Ingestion Adapter

Status: Accepted

## Context

The Portuguese food catalogue is sourced from the INSA *Base de Dados da
Composição de Alimentos* (BDCA) v7.1 (2026), published via PortFIR. An initial
integration (commit `db7bfe6`, PR #5) already ships all 1,376 foods as
**approved** seed data, but it carries only two nutrients (available
carbohydrate `CHOAVL` and energy `ENERC`) and was generated by a throwaway
`scratchpad/gen_insa.py` script with no provenance preservation, quality gates,
or reproducibility guarantees.

The workbook actually contains **48 nutrients** per food, a 3-level FoodEx2
Portuguese classification, and per-food basis information (per 100 g edible,
except alcoholic beverages per 100 ml). INSA's terms grant **use with mandatory,
visible attribution**; they do **not** explicitly address commercial-scale reuse
or redistribution of the raw dataset.

## Decision

1. **Keep INSA published in place** as `sourceId: "INSA-BDCA"`, `pt-insa-<code>`
   ids, `status: "approved"`, licence `insa-portfir-attribution`. Do not regress
   the shipped catalogue.
2. **Replace the throwaway generator** with a proper, source-specific ingestion
   adapter under `services/food-ingestion/` (Python, stdlib + openpyxl), matching
   the design its README already prescribes: immutable snapshot + SHA-256,
   structural validation, raw source-record preservation, a curated 48-nutrient
   mapping table, quality checks with quarantine, deterministic/idempotent runs,
   and version-to-version diffing.
3. **Expand the canonical model additively** (no breaking change): add the `µg`
   nutrient unit, an optional `SourceReference.attribution`, and optional
   `CanonicalFood.preparationState` / `foodGroup` / `ediblePortion` / `density`,
   plus an in-repo nutrient dictionary (`@t1dine/food-schema`).
4. **Regenerate the committed catalogue** (`portugalInsa.ts` via `insaBuilder.ts`)
   from the adapter with all 48 mapped nutrients, food group, and preparation
   state. The mapping is a curated deterministic table — **no generative AI** is
   involved in nutrient values, and the dose engine is untouched.
5. **Do not commit or redistribute the raw workbook.** Only derived per-basis
   values are committed, always with attribution. The raw file stays git-ignored
   (`docs/data/*.xlsx`); its SHA-256 is recorded.

## Nutrient semantics (verified from the workbook's own correspondence sheet)

INSA "Hidratos de carbono" is **available carbohydrate** (INFOODS `CHOAVL`), not
carbohydrate-by-difference, with fibre (`FIBTG`) tracked separately. This is the
exact carbohydrate T1Dine's meal maths already keys on, so the dose-relevant
value maps 1:1. Missing cells are preserved as *missing* (observation omitted) —
never coerced to zero. The source has no trace/below-detection tokens, so trace
and true zero are indistinguishable in the data; this limitation is documented.

## Consequences

- Portugal gains a rich, provenance-preserving national composition source, and
  future BDCA versions can be re-ingested deterministically and diffed.
- Two open licence questions remain recorded in
  `docs/data/portfir-source-governance.md`: commercial-scale reuse and raw
  redistribution. These do not block *use with attribution*, which the terms
  grant, but must be resolved before any raw-dataset redistribution.
- Dose Assist isolation, provenance rules (ADR-0004), and the offline-first
  contract (ADR-0002) are unaffected.
